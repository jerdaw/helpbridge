# KCC vs 211 Positioning Playbook (Evidence-Aligned)

**Date:** February 27, 2026  
**Use:** External copy, partner conversations, internal alignment

## Positioning core

Use this one-liner:

KCC is a Kingston-first, privacy-first digital access layer that complements 211’s broader network and live support channels.

## What to say

1. "We focus on Kingston depth and low-friction discovery."
2. "Search is privacy-preserving by design, and we do not log query text."
3. "We support offline-capable access for low-connectivity contexts."
4. "We complement 211 by improving local digital access, then guiding people to broader supports when needed."

## What not to say

1. "We replace 211."
2. "Every listing is manually verified."  
   Use instead: "Listings are verification-tiered and we publish verification provenance."
3. "No analytics at all."  
   Use instead: "No third-party tracking and no query-text logging."
4. "More languages than 211."  
   Use instead: "Our web UI supports 7 locales; 211 provides broader language access through live channels."

## Approved comparison language

### Safe

1. "211 provides broad provincial/national coverage and live navigation."
2. "KCC provides a locally optimized digital interface with privacy-first search and offline capability."
3. "The services are complementary: breadth and live triage from 211, local digital usability from KCC."

### Avoid

1. "KCC is more accurate than 211."
2. "KCC is the best way for everyone in crisis."
3. "KCC has stronger language coverage than 211."

## Proof points to keep current

1. Verification freshness metrics:
   - count by verification level,
   - count pending manual review,
   - median days since verification.
2. Access outcomes:
   - time to first relevant result,
   - successful call click-through/contact rates,
   - crisis handoff latency.
3. Privacy posture:
   - query logging status,
   - analytics payload schema boundaries,
   - third-party script inventory.
4. Accessibility status:
   - latest CI accessibility run,
   - unresolved severity-ranked issues.

## Escalation trigger

Pause comparative claims immediately if any of these are true:

1. Verification provenance cannot substantiate "manual" statements.
2. Accessibility test regressions are unresolved in release candidates.
3. Privacy copy claims "no analytics" while analytics events remain active.

## Partner conversation template

1. Start with complementarity:
   - "211 is foundational for broad coverage and live navigation."
2. State KCC lane:
   - "KCC optimizes local digital access, privacy, and offline resilience."
3. Offer collaboration:
   - "We want interoperable referral and data-quality loops, not duplicate systems."
